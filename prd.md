# PRD: «Умная корзина» — оптимизатор семейных закупок (MVP)

**Версия:** 0.1 (MVP для тренинга по вайбкодингу)
**Дата:** 09.07.2026
**Автор:** Alex
**Статус:** Draft

---

## 1. Проблема

Семья еженедельно закупает потребительские товары (продукты питания, бытовая химия, товары для ребёнка). Цены на один и тот же товар в разных типах торговых точек Алматы различаются на 10–40%: овощи и мясо дешевле на базаре, бытовая химия — в гипермаркете, часть позиций выгоднее заказать онлайн. Сейчас покупатель либо переплачивает, покупая всё в одном месте, либо тратит время на ручное сравнение цен по памяти.

## 2. Цель продукта

Дать пользователю возможность **перед походом за покупками** составить список товаров и получить оптимальное распределение этого списка по торговым точкам с расчётом экономии.

## 3. Целевая аудитория

Родители / взрослые члены семьи, регулярно совершающие закупки для домохозяйства в Алматы. MVP — single user, без авторизации.

## 4. Пользовательская история (основная)

> **Как** родитель, планирующий еженедельные закупки для семьи,
> **я хочу** видеть сравнение цен на товары из моего списка в разных типах точек (гипермаркет, базар, магазин у дома),
> **чтобы** разбить список на оптимальные корзины по точкам и не переплачивать.

## 5. Скоуп MVP

### Входит (In Scope)

1. **Тестовый справочник товаров** — статический JSON-файл, ~20 позиций в 3 категориях:
   - Продукты питания (мясо, овощи, молочные продукты, крупы, хлеб)
   - Бытовая химия (стиральный порошок, средство для посуды, туалетная бумага)
   - Товары для ребёнка (подгузники, детское питание)
2. **Три типа торговых точек** с ценами по каждому товару:
   - Гипермаркет (референс: Magnum)
   - Базар (референс: Зелёный базар / Алтын Орда)
   - Магазин у дома
3. **Составление списка покупок**: чекбоксы + указание количества (кг/шт/л).
4. **Расчёт оптимального распределения**: для каждого выбранного товара определяется точка с минимальной ценой.
5. **Итоговая сводка**:
   - Три корзины (что купить в каждой точке) с подытогом.
   - Общая сумма при оптимальном распределении.
   - Сумма при сценарии «всё в гипермаркете» (базовый сценарий).
   - **Экономия** = разница между сценариями (в тенге и %).
6. **Порог целесообразности**: настройка «не отправляй меня в точку ради экономии меньше N тенге» (по умолчанию 1000 ₸). Если корзина точки даёт экономию ниже порога — её товары переносятся в основную точку.

### Не входит (Out of Scope — итерации 2+)

- Реальные данные из API (data.egov.kz, stat.gov.kz) и парсинг ритейла (Magnum, Arbuz, Kaspi).
- История цен и графики динамики.
- Учёт стоимости дороги и времени.
- Авторизация, сохранение списков между сессиями, мультипользовательность.
- Мобильное приложение (MVP — адаптивная веб-страница).

## 7. Нефункциональные требования

- **Стек:** одностраничное веб-приложение (React или vanilla HTML/JS — на усмотрение инструмента), без бэкенда, без БД.
- **Данные:** справочник — встроенный JSON-объект в коде или отдельный файл `data.json`.
- **UI:** один экран; слева/сверху — справочник с выбором, справа/снизу — результат. Адаптив для мобильного.
- **Язык интерфейса:** русский. Валюта: тенге (₸).
- **Производительность:** пересчёт корзин мгновенный (при каждом изменении выбора).

## 9. Логика расчёта (алгоритм)

1. Для каждого выбранного товара: `cost[store] = price[store] × quantity`.
2. Оптимальная точка товара = `argmin(cost)`.
3. Сформировать корзины: сгруппировать товары по оптимальным точкам.
4. `baseline` = сумма всех товаров по ценам гипермаркета.
5. `optimal` = сумма всех товаров по минимальным ценам.
6. `savings = baseline − optimal`.
7. Применить порог: для каждой неосновной корзины посчитать её вклад в экономию (разница между её стоимостью в своей точке и в гипермаркете); если вклад < порога — перенести товары корзины в гипермаркет и пересчитать.

## 10. Критерии приёмки MVP

- [ ] Пользователь выбирает ≥ 1 товар и видит результат без перезагрузки страницы.
- [ ] Результат содержит: состав каждой корзины, подытог по корзине, общую сумму, экономию в ₸ и %.
- [ ] Изменение количества товара немедленно пересчитывает результат.
- [ ] Порог целесообразности влияет на распределение (проверяется на кейсе: одна точка даёт экономию 300 ₸ при пороге 1000 ₸ → товары уходят в гипермаркет).
- [ ] Приложение открывается и работает на мобильном экране (390px).

## 11. Метрика успеха MVP (в рамках тренинга)

Работающий прототип, демонстрирующий полный цикл «выбрал товары → получил разбивку по точкам → увидел экономию» за одну сессию вайбкодинга.
